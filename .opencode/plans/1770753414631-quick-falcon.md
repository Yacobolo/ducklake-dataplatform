# Plan: Add Go binary artifacts to .gitignore

## Problem
Running `go build` and `go test -c` produces binary files in the root directory (`server`, `server_test`, both ~73MB). These are untracked but not ignored, cluttering `git status`.

## Changes

**File:** `.gitignore`

Add a broad pattern that ignores all extensionless files at the root (which catches Go binaries), with an exception for `Makefile`:

```gitignore
.opencode/
.env
ducklake_meta.sqlite

# Ignore extensionless files at root (Go build/test binaries)
/*
!/*.*
!/Makefile
```

### How it works
- `/*` — ignore all files at the root level
- `!/*.*` — un-ignore files that have an extension (`.go`, `.mod`, `.yaml`, etc.)
- `!/Makefile` — un-ignore `Makefile` (extensionless but needs to be tracked)

Note: Dotfiles (`.gitignore`, `.env`, etc.) are not matched by `/*` so they are unaffected.

### Root directory files affected
| File | Has extension? | Result |
|---|---|---|
| `Makefile` | No | Kept via `!/Makefile` exception |
| `server` | No | **Ignored** |
| `server_test` | No | **Ignored** |
| `go.mod`, `go.sum`, `sqlc.yaml`, etc. | Yes | Kept via `!/*.*` |
| `.gitignore`, `.env` | Dotfiles | Unaffected |

## Verification
- Run `git status` — confirm `server` and `server_test` no longer appear as untracked
- Confirm `Makefile` is still tracked: `git ls-files Makefile`
